# Ink Echo - モバイルファースト ワイヤーフレーム & 画面設計

作成日: 2026年2月  
対象: モバイル縦持ち（iPhone/Android 標準サイズ想定、約375px幅）  
デザイン方針:  
- 極力ミニマル（デジタルをシンプルに保つ哲学に基づく）  
- 低忠実度（low-fidelity）  
- 主要アクション（マイク、アップロード）は画面下部に配置  
- タッチターゲットは指で押しやすい大きさ（最低44×44dp）  

## 画面遷移フロー概要
```
スタート
↓
メイン対話画面（録音待機）
↓ (7ターン繰り返し)
7ターン終了オーバーレイ
↓
写真アップロード画面 / オーバーレイ
↓ (アップロード → Mistral Vision解析)
メイン対話画面に戻る（再開）
↓ (セッション内繰り返し可能)
```

## 1. メイン対話画面（録音待機状態）
```
┌──────────────────────────────────────┐
│ Ink Echo                  ×         │  ← 薄いヘッダー（固定）
└──────────────────────────────────────┘
┌──────────────────────────────────────┐
│                                      │
│  AI: こんにちは。                    │  ← 会話ログ（上から古い順）
│      今どんなことを考えていますか？  │
│      ▶                               │  ← 各AIメッセージに再生ボタン
│                                      │
│  あなた: 最近モチベーションが…      │
│                                      │
│  AI: それは「やる気が出ない」のか、  │
│      「やるべきことが不明確」なのか  │
│      どちらに近いですか？            │
│      ▶                               │
│                                      │
│  （ログはここまでスクロール）        │
│                                      │
└──────────────────────────────────────┘
↑ スクロール可能 ↑
┌─────────────────────┐
│        ● 話す       │  ← 大きなマイクボタン（中央下固定）
│     (録音開始)      │
└─────────────────────┘
```
- マイクボタン：赤で録音中強調、録音時は波形アニメーションに変化
- 会話ログ：吹き出し風（簡易四角＋テキスト）でも可
- 再生ボタン：▶アイコン、タップで音声再再生

│                                      │

│                                      │
│      今、ここで重要な気づきを        │

│                                      │

│      アップロードしてね。            │
│      （それがなければ続きはないよ）  │
│                                      │
│        ┌─────────────────────┐       │
│        │  写真をアップロード  │  ← 大きく目立つボタン（青系推奨）
│        └─────────────────────┘       │

- この時点で固定メッセージが自動音声再生される
- 閉じる×は「セッションを破棄」扱い（再開不可）

## モバイルファースト設計のポイント

- 縦長スクロール前提（横持ち非対応でも問題なし）
- 主要アクション（マイク・アップロード）は画面下部1/3以内に配置
- タッチターゲットサイズ：最低44×44dp以上
- 文字サイズ：本文16–18px、ボタンラベル18–20px
- 色使い：極力控えめ（グレー/黒基調、アクセントは青・赤のみ）
- アクセシビリティ：VoiceOver対応（ボタンにaria-label必須）

## 備考・実装時の注意

- オーバーレイは半透明グレー（backdrop-filter: blur推奨）
- 録音中は画面ロック防止（Wake Lock API検討）
- プレビュー写真は圧縮表示（ファイルサイズ軽減）
- エラー時は声で「もう一度話しかけて」と優しく返す
